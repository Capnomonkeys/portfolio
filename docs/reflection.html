Reflective Essay

In developing a website, the first thing I discovered is just how easy it was. The next thing I found out, of course, is how difficult and aggravating it all is. It's a strange tonal shift you discover about when you want to make a website that isn't actually just bland text on a screen, but it certainly isn't insurmountable. I almost see it as a puzzle, not just to get the website actually functioning through code, but also in attempting to figure out what exactly a website should look like. Given that making a simple website and uploading it is fairly easy however, I would say that the creative design and application of CSS is the most difficult thing to figure out for a new student.

Despite that, I found Wordpress to be the more annoying website to deal with. It's a tad ironic, given both the fact that it's intended for a wider audience, and that I'm typing this essay out on the site, but you can certainly tell that it's intended for blogging first, full creative website design second. Using HTML and CSS directly allows for a much greater degree of freedom in designing your website however you want, even changing the fundamental design from page to page (if you want, probably not the best idea overall...) can be done fairly easily once you learn all the basic commands, tags, and controls. Wordpress, on the other hand, intends for you to interact with it in a very specific way. You're going to write, maybe insert some images, and you'll put it all on a standardized format that can be tweaked to a degree that it inspires creativity that's beyond the restraints put upon you by the base website design. Essentially, Wordpress works if you just have text you want to put out into the open web, but full on HTML + CSS is better for those that want a fully customizable, unique site to call their own.

For both websites, however, you will need a healthy dose of creativity to actually make something. You need to be able to write something at least partially interesting, and you need to have an eye for basic text formatting else everything ends up in a giant, unreadable chunk. There also needs to be some sort of graphical design insight, even if it's just copying an existing site's design for the most part. An aesthetically pleasing site can go a long way in encouraging people to stay and read the things you've written. You also should have at least a passing knowledge of HTML and CSS, obviously for the websites entirely made out of those languages, but also for Wordpress since you can use the code view and implement some outside CSS for the main website shell.

For me personally, I want to get a better eye for graphical designs, but I think everything I do can be improved in some way. It's definitely in a more "make more websites and pages and experiment more" sort of way though, I don't think I'm unable to make a passable website right now or anything. Really, I just need more experience in website design to truly feel like I can make something great.
